## Candidate argument

Product advantage begins before the mission window opens. The campaign proposes a shared Mission Window Contract so portfolio decisions connect mission pull, human authority, integration fit, evidence, acquisition path, and field learning without flattening product-specific work.

## Public artifact set

- `index.html` — responsive candidate vision and Mission Window Synchronizer
- `resume.html` — two-page role-aligned resume
- `cover-letter.html` — one-page cover letter
- `interview-brief.html` — interview thesis brief
- `120-day-plan.html` — entry plan
- `mission-window-review.html` — portfolio decision artifact
- `docs/` — generated printable PDFs
- `brand-intelligence.md` and `brand-tokens.css` — identity provenance and implementation record
- `qa/` — source and rendered regression checks

## Brand and authorship

The locally stored Two Six Technologies mark is used only to identify the target employer. The campaign is clearly labeled as independent candidate work and does not imply company sponsorship, approval, or access to undisclosed systems.

## Publication

The GitHub Pages workflow runs the campaign regression audit, rendered browser review, PDF generation, and site deployment from `main`. Generated documents and the deployment receipt are persisted after a successful release.
