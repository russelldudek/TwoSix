# Accessibility Audit

- Skip link targets the primary content.
- Official company mark includes meaningful alternative text in the header.
- Primary navigation is labeled and responsive disclosure state uses `aria-expanded` and `aria-controls`.
- Product scenarios use tab semantics, selected-state reporting, one active tab stop, pointer input, and Arrow/Home/End keyboard behavior.
- Decision output uses a polite live region.
- Focus indicators are visible on links, buttons, and summary controls.
- Essential information remains available without interaction.
- Reduced motion preserves every label, disposition, and scenario result.
- Mobile does not require internal horizontal scrolling.
- Document actions remain keyboard accessible and are removed from print output.

Rendered accessibility behavior is included in the browser quality workflow.
