# Role-Derived Motion Contract

The motion represents a portfolio decision resolving through mission pull, human authority, integration fit, and acquisition path.

- Trigger: selecting Helix, Sentr, or SIGMA.
- Intermediate state: the board enters `is-running`; the decision signal crosses the desktop gate system or the handheld gate cards resolve in sequence.
- State transaction: labels, geometry, colors, disposition, evidence posture, selected tab, and tab stop update together.
- Settlement: the animation class is removed after 1.2 seconds and the final state remains stable.
- Rapid selection: the prior timer is cleared before the next transition; the final request remains authoritative.
- Replay: selecting any scenario again replays its transition.
- Reduced motion: the complete semantic result appears immediately with no transition.

The motion is explanatory, finite, keyboard operable, and never required to access the underlying information.
