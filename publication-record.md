# Publication Verification Record

Live candidate vision: https://russelldudek.github.io/TwoSix/

The release workflow must:

1. run source regression;
2. render the full route and viewport matrix;
3. generate every printable document from HTML;
4. verify exact PDF contracts and content boundaries;
5. assemble the complete static release;
6. deploy that exact source commit to GitHub Pages; and
7. persist generated PDFs plus `deployment-status.json` only after success.

A successful source commit alone is not live proof. Completion requires the deployment receipt to identify the corrected source commit and the public candidate-vision URL.
